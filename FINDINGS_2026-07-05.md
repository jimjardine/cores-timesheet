## Headline: SMS intake is healthy

The urgent issue is resolved. Deploying `sms-timesheet` via the Supabase MCP had silently forced `verify_jwt=true` on v21, which would have rejected every Twilio webhook call. Redeployed as **v22 with `verify_jwt: false`** (identical code, flag only) and live-tested: an unauthenticated Twilio-style form POST returns HTTP 200 TwiML. Edge-function logs show **zero invocations during the broken window — no worker texts were lost**.

> ⚠️ Standing rule: never deploy this function via MCP; use the CLI with `--no-verify-jwt`.
